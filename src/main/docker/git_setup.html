Frinex GIT Configuration

The Frinex build service accepts experiment configuration files via GIT repositories. The repository cannot be accessed via a web browser, to access your assigned repository you will need to use a GIT client.

There are many GIT clients to choose from, you can use the GIT command line application or a GIT GUI application access your Frinex repository.

One such tool is Visual Studio Code, which has the advantage of providing an XML editor and a GIT client in the same tool. If you chose to use Visual Studio Code then you might also want to use the XML Language Support by Red Hat extension. The XML extension will validate your XML but can also incorporate the Frinex documentation into that validation and provide help in the form of context based documentation and code completion.

Whichever tool you choose to edit your XML you will want to ensure that the Frinex XSD is correcty specified at the start of your XML document. You can find an example of this in the minimal_example.xml. The XML documentation can also be viewed directly in the Frinex Markup document.
